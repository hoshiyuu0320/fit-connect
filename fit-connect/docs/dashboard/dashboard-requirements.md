# ダッシュボード機能 要件定義

## 📋 目的

トレーナーが**1日の業務を効率的に開始・管理できる**ための「コックピット」画面を提供する。

---

## 🎯 表示情報の詳細

### 1. 一目で分かる重要数値（KPI）

**目的:** トレーナーの業務状況を数値で可視化し、現状を即座に把握できるようにする

#### 表示項目

| 項目名 | データソース | 計算ロジック | 優先度 |
|--------|------------|------------|--------|
| 担当顧客数 | `clients` テーブル | 現在のトレーナーIDに紐づく顧客の総数 | 高 |
| 今週の新規メッセージ数 | `messages` テーブル | 過去7日以内に受信したメッセージ数（`receiver_id = trainer_id`） | 高 |
| アクティブ顧客数 | `weight_records`, `meal_records`, `exercise_records` | 過去30日以内にいずれかの記録がある顧客数 | 中 |
| 要対応チケット数 | `tickets` テーブル | 14日以内に期限切れ予定のチケット数 | 中 |

#### UIレイアウト

```
┌────────────┬────────────┬────────────┬──────────────────┐
│ 担当顧客数 │ 今週のメッセ│ アクティブ │ 要対応チケット   │
│   24人     │  ージ: 18件 │  顧客: 20人│    3件           │
└────────────┴────────────┴────────────┴──────────────────┘
```

**デザインガイドライン:**
- カード形式で表示
- 各KPIにアイコンを配置（👥, 💬, ✅, ⏰）
- 数値は大きく、ラベルは小さく
- hover時に詳細ツールチップ表示（将来実装）

---

### 2. 最近のアクティビティ（メッセージプレビュー）

**目的:** 返信が必要なメッセージを見逃さず、迅速に対応できるようにする

#### 表示項目

- **表示件数:** 直近5件
- **対象メッセージ:** トレーナーが受信したメッセージ（`receiver_id = trainer_id`）
- **並び順:** 新しい順（`timestamp DESC`）

#### 各メッセージの表示内容

| フィールド | 表示形式 | 備考 |
|-----------|---------|------|
| 顧客名 | `clients.name` | プロフィール画像も表示（将来） |
| メッセージ本文 | 最大50文字 | 50文字以上は「...」で省略 |
| 送信時刻 | 相対時刻 | 「2分前」「1時間前」「11/8 15:30」 |

#### UIレイアウト

```
┌─────────────────────────────────────────────────┐
│ 💬 最近のメッセージ                    [全て見る] │
├─────────────────────────────────────────────────┤
│ 👤 山田太郎さん                        2分前     │
│    「明日のセッション、15時でも大丈夫ですか？」 │
├─────────────────────────────────────────────────┤
│ 👤 佐藤花子さん                        1時間前   │
│    「今日の食事記録送りました！」              │
└─────────────────────────────────────────────────┘
```

**インタラクション:**
- 各メッセージをクリック → `/message?clientId=xxx` に遷移
- 「全て見る」ボタン → `/message` に遷移

---

### 3. 注目すべき顧客（アラート）

**目的:** フォローが必要な顧客を自動的に検出し、適切なタイミングで介入できるようにする

#### アラート条件

| アラート種別 | 条件 | 優先度 | 表示色 |
|------------|------|--------|--------|
| 記録なし顧客 | 7日以上、体重・食事・運動のいずれも記録がない | 高 | 🔴 赤 |
| チケット期限切れ間近 | 残回数が3回以下 かつ 期限が14日以内 | 中 | 🟡 黄 |
| 体重目標乖離 | 最新の体重が目標体重から±2kg以上乖離 | 中 | 🟡 黄 |

#### UIレイアウト

```
┌─────────────────────────────────────────────────┐
│ ⚠️  要確認                                      │
├─────────────────────────────────────────────────┤
│ 🔴 鈴木一郎さん - 7日間記録なし                │
│ 🟡 田中美咲さん - チケット残り2回（期限: 5日後）│
│ 🟡 高橋健太さん - 体重が目標から+2kg           │
└─────────────────────────────────────────────────┘
```

**インタラクション:**
- 各アラートをクリック → `/clients/[client_id]` に遷移

---

### 4. クイックアクション（よく使う機能）

**目的:** 頻繁に使う機能に1クリックでアクセスし、業務効率を向上させる

#### アクションボタン

| ボタン名 | 遷移先 | アイコン | 優先度 |
|---------|--------|---------|--------|
| 顧客を探す | `/clients` | 🔍 | 高 |
| メッセージ | `/message` | 💬 | 高 |
| レポート | `/report` | 📊 | 中（将来実装） |
| スケジュール | `/schedule` | 📅 | 中（将来実装） |

#### UIレイアウト

```
┌──────────────────────────────────────────────────────────┐
│ ⚡ クイックアクション                                     │
│  [🔍 顧客を探す]  [💬 メッセージ]  [📊 レポート]  [📅 スケジュール] │
└──────────────────────────────────────────────────────────┘
```

**デザインガイドライン:**
- ボタンは横並びで配置
- hover時に背景色を変更
- 実装済みと未実装で視覚的に区別（未実装はグレーアウト）

---

### 5. 本日の予定（将来実装）

**目的:** 今日のセッションスケジュールを一覧表示し、準備を効率化する

#### 表示条件

- `sessions` テーブルから本日のセッションを取得
- ステータスが「予定」または「確定」のもののみ表示
- 時刻順に並び替え

#### 表示項目

- セッション時刻（開始〜終了）
- 顧客名
- セッション内容（メモ欄）

#### UIレイアウト

```
┌─────────────────────────────────────────────────┐
│ 📅 本日の予定 (3件)                             │
├─────────────────────────────────────────────────┤
│ 10:00 - 11:00  山田太郎さん（筋トレ）          │
│ 14:00 - 15:00  佐藤花子さん（ヨガ）            │
│ 18:00 - 19:00  鈴木一郎さん（パーソナル）      │
└─────────────────────────────────────────────────┘
```

**注意:** この機能は`sessions`テーブル作成後に実装

### レイアウト仕様

- **グリッドシステム:** 2カラムレイアウト（メッセージ:アラート = 1:1）
- **レスポンシブ:** タブレット以下では1カラムに切り替え
- **余白:** セクション間は6（24px）、カード内は4（16px）
- **カラースキーム:** 既存のTailwind CSSテーマに準拠

## 🎯 成功指標（KPI）

実装後、以下の指標でダッシュボードの有効性を測定する：

1. **ページ滞在時間:** 平均2-3分（情報の確認に十分な時間）
2. **クイックアクションの利用率:** 50%以上のユーザーが1日1回以上利用
3. **メッセージ応答時間:** 平均応答時間が30%短縮
4. **アラート対応率:** 要確認アラートに対して80%以上が24時間以内に対応
