# FIT-CONNECT 要件定義書（PRD）

## 1. プロジェクト概要

### 1.1 プロジェクト名
**FIT-CONNECT** - パーソナルトレーナー向け顧客管理システム

### 1.2 プロジェクトの目的
パーソナルトレーナーの業務効率化と、顧客の健康・フィットネスデータの一元管理を実現するシステム。
トレーナーと顧客間のコミュニケーションを円滑にし、顧客の目標達成をサポートする。

### 1.3 ターゲットユーザー

| ユーザー種別 | 説明 | 利用デバイス |
|-------------|------|-------------|
| **トレーナー** | パーソナルトレーナー、フィットネスインストラクター | PC/タブレット（Webアプリ） |
| **顧客（クライアント）** | トレーナーの指導を受ける一般ユーザー | スマートフォン（ネイティブアプリ） |

### 1.4 ビジネス目標
1. トレーナーの業務効率を**30%以上**向上
2. 顧客の継続率（リテンション）を**20%以上**改善
3. 顧客とトレーナー間のコミュニケーション頻度を**2倍**に増加

## 4. 非機能要件

### 4.1 パフォーマンス
| 項目 | 要件 | 備考 |
|------|------|------|
| ページ読み込み | 3秒以内 | 初期ロード |
| API応答時間 | 500ms以内 | 95パーセンタイル |
| リアルタイム遅延 | 1秒以内 | メッセージング |

### 4.2 セキュリティ
| 項目 | 要件 | 備考 |
|------|------|------|
| 認証 | Supabase Auth | セッション管理 |
| データアクセス制御 | Row Level Security (RLS) | テーブル単位 |
| 通信暗号化 | HTTPS必須 | TLS 1.2以上 |
| 個人情報保護 | 暗号化保存 | 顧客データ |

### 4.3 可用性
| 項目 | 要件 | 備考 |
|------|------|------|
| 稼働率 | 99.5%以上 | 月間 |
| バックアップ | 日次 | Supabase自動バックアップ |
| 障害復旧 | 4時間以内 | RTO |

### 4.4 スケーラビリティ
| 項目 | 要件 | 備考 |
|------|------|------|
| 同時接続ユーザー | 1,000人以上 | 将来目標 |
| データ保持期間 | 無制限 | 記録データ |
| ストレージ | 自動拡張 | Supabase Storage |

## 5. データモデル

### 5.1 主要エンティティ

```
profiles (トレーナー)
    ├── clients (顧客) 1:N
    │       ├── weight_records (体重記録) 1:N
    │       ├── meal_records (食事記録) 1:N
    │       ├── exercise_records (運動記録) 1:N
    │       ├── tickets (チケット) 1:N
    │       └── sessions (セッション) 1:N
    ├── messages (メッセージ) 1:N
    └── sessions (セッション) 1:N
```

### 5.2 テーブル一覧

| テーブル名 | 説明 | 主なカラム |
|-----------|------|-----------|
| profiles | トレーナー情報 | id, name |
| clients | 顧客情報 | client_id, trainer_id, name, gender, age, height, target_weight, purpose |
| messages | メッセージ | id, sender_id, receiver_id, sender_type, receiver_type, message |
| sessions | セッション予定 | id, trainer_id, client_id, session_date, status, session_type |
| weight_records | 体重記録 | id, client_id, weight_kg, recorded_at |
| meal_records | 食事記録 | id, client_id, meal_type, calories, images, recorded_at |
| exercise_records | 運動記録 | id, client_id, exercise_type, duration_minutes, calories_burned |
| tickets | チケット | id, client_id, total_sessions, remaining_sessions, valid_until |

詳細は [db-design.md](./db-design.md) および [er-design.md](./er-design.md) を参照。
