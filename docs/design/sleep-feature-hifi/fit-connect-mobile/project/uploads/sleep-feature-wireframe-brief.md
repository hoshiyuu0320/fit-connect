# 睡眠データ連携 — UIワイヤーフレーム生成ブリーフ

Claude Design 等のUIデザインエージェントに投入してワイヤーフレーム・モックアップを生成させるための構造化ブリーフ。

---

## 1. プロジェクトコンテキスト

| 項目 | 値 |
|---|---|
| プロジェクト | FIT-CONNECT Mobile（クライアント向けフィットネス管理アプリ） |
| 対象ユーザー | トレーナーと契約している一般利用者（クライアント） |
| プラットフォーム | iOS / Android（Flutter） |
| 機能 | 睡眠データ連携（HealthKit / Health Connect + 簡易手動入力） |
| デザインコンセプト | Modern, Sophisticated, Gender-neutral（健康管理ツール） |
| 既存パターンの参照元 | `lib/features/weight_records/presentation/screens/weight_record_screen.dart`（体重記録画面。統一感を保つこと） |

**睡眠ステージ配色（新規定義）**:

| ステージ | 色 | 由来 |
|---|---|---|
| 深い睡眠 | `#4338CA` (indigo-700) | 濃い紫青 |
| 浅い睡眠 | `#818CF8` (indigo-400) | 中間の紫青 |
| REM睡眠 | `#60A5FA` (primary400) | ブルー |
| 覚醒 | `#E2E8F0` (slate200) | グレー |

### タイポグラフィ

- フォントファミリ: **Noto Sans JP**（全ての日本語テキスト）
- 時間表示（大きな数値）: `font-weight: 700`、`font-size: 28-32px`、`letter-spacing: -0.02em`
- 本文: `font-size: 14-16px`、`font-weight: 400`
- ラベル / キャプション: `font-size: 12-13px`、`font-weight: 500`、色は `textSecondary`

### 角丸

- カード: **8px**（max）
- ボタン: **6px**
- タグ/チップ: **6-8px**
- 禁止: 16px以上の過度な丸み、完全円形ボタン（FAB除く）

### スペーシング

- カード内パディング: **16px**
- セクション間マージン: **24px**
- リストアイテム間: **12px**

### アンチパターン（禁止事項）

- AI特有の派手なグラデーション
- 濃いドロップシャドウ（`shadow: 0x0D000000`（黒5%）程度に抑える）
- 彩度の高い原色（ステータス以外で使用禁止）
- 要素の詰め込み過多

### アイコン

- ライブラリ: **Lucide Icons**（`lucide_icons_flutter` パッケージ）
- 主要アイコン:
  - 睡眠: `LucideIcons.moon`
  - HealthKit由来: `LucideIcons.heartPulse`
  - 手動入力: `LucideIcons.penLine`
  - 編集: `LucideIcons.pencil`
  - 目覚め評価（カスタム絵文字 or アイコン）:
    - すっきり: 😊 または `LucideIcons.smile`（色: `success`）
    - まあまあ: 😐 または `LucideIcons.meh`（色: `warning`）
    - だるい: 😫 または `LucideIcons.frown`（色: `error`）

## 4. コンポーネント詳細

### C1. SleepSummaryCard（ホーム画面の睡眠サマリー）

**サイズ**: 横幅いっぱい、縦幅 96-120px

**状態バリエーション（3種類 生成必須）**:

#### 4-1-a. 状態「記録あり（HealthKit連携）」
```
レイアウト:
┌────────────────────────────────────┐
│  🌙 昨夜の睡眠            🫀         │  ← 左: moonアイコン+ラベル / 右: healthPulseアイコン（14px）
│                                    │
│  7時間23分         😊 すっきり       │  ← 左: 時間大きく / 右: 目覚め評価
│  23:45 → 7:08                      │  ← 就寝→起床時刻（小さく）
└────────────────────────────────────┘

タイポ:
- 「昨夜の睡眠」: 12px, weight 500, textSecondary
- 「7時間23分」: 24px, weight 700, textPrimary
- 「23:45 → 7:08」: 12px, weight 400, textHint
- 「すっきり」: 14px, weight 600, success色

背景: cardBackground (#FFFFFF), border 1px slate100
角丸: 8px
```

#### 4-1-b. 状態「記録あり（手動のみ / 目覚め評価のみ）」
```
┌────────────────────────────────────┐
│  🌙 昨夜の睡眠            ✏️         │  ← 右アイコンが penLine に変わる
│                                    │
│  😊 すっきり                        │  ← 時間表示は無し、評価のみ大きめ
│  タップして詳細を記録                 │  ← 補足テキスト
└────────────────────────────────────┘
```

#### 4-1-c. 状態「未記録（CTA表示）」
```
┌────────────────────────────────────┐
│  🌙 昨夜の睡眠                       │
│                                    │
│  [ 目覚めを記録する → ]              │  ← primary色のアウトラインボタン、横幅いっぱい
└────────────────────────────────────┘

ボタン:
- 高さ: 44px
- 背景: primary50
- 前景: primary
- 角丸: 6px
- アイコン: 矢印（arrowRight）
```

**インタラクション**:
- カード全体タップ → SleepRecordScreen へ遷移
- 未記録時のCTAボタンタップ → WakeupRatingSelector をボトムシートで表示（画面遷移なし）

### S1. SleepRecordScreen（専用画面）

**構成**: 縦スクロール1画面（タブ不使用）

**AppBar**:
- タイトル: 「睡眠記録」
- 戻るボタン: 左上
- 右上アクション: 「同期」アイコンボタン（`LucideIcons.refreshCw`、HealthKit連携ON時のみ）

**セクション順**:

#### S1-A. 昨夜のサマリーカード（最上部）

```
┌────────────────────────────────────┐
│ 🌙 昨夜の睡眠                  🫀 HealthKit │
│                                    │
│  7時間23分                          │  ← 大きく（32px, weight 700）
│  ─────────────                    │
│  就寝 23:45    起床 7:08            │  ← 2カラム、ラベル+時刻
│                                    │
│  ステージ（円グラフ or 水平バー）      │
│  ─────────────                    │
│  ■ 深 25% (1h50m)                  │
│  ■ 浅 50% (3h41m)                  │
│  ■ REM 20% (1h28m)                 │
│  ■ 覚 5% (22m)                     │
│                                    │
│  ─────────────                    │
│  目覚め：😊 すっきり  [編集]          │  ← 右端に「編集」テキストボタン
└────────────────────────────────────┘
```

**手動のみの状態**: ステージ・時刻情報を非表示、「目覚め」のみ大きく表示。「詳細データを取得するにはヘルスケア連携を有効にしてください」の案内を下部に小さく表示。

**未記録の状態（当日データ無し）**:
```
┌────────────────────────────────────┐
│  🌙 今日の記録はまだありません         │
│                                    │
│  [ 目覚めを記録する ]                │
└────────────────────────────────────┘
```

#### S1-B. 週間トレンド

**タイトル**: 「直近7日間」（左寄せ、14px weight 600）

**グラフ**: `fl_chart` LineChart
- X軸: 日付（4/12, 4/13, ...）
- Y軸: 睡眠時間（時間単位、4-10h レンジ）
- 線色: `primary`、丸いノード、下部にうっすらグラデ（`primary50` からの透明）
- データなしの日は点線でスキップ

**目覚め評価の補助表示**: 各ノードの下に小さく 😊/😐/😫 アイコン（オプション、混雑感が出るなら省略）

**高さ**: 160-180px

#### S1-C. 履歴リスト

**タイトル**: 「履歴」

**1アイテムのレイアウト**:
```
┌────────────────────────────────────┐
│ 4/18(土)  7h23m      😊  🫀     →  │  ← 日付 / 時間 / 目覚め / ソース / chevron
└────────────────────────────────────┘

- 日付: 14px weight 600, textPrimary
- 時間: 16px weight 700, textPrimary（手動のみの場合は「--」）
- 目覚めアイコン: 20x20px
- ソースアイコン: healthPulse(12px, slate500) or penLine(12px, slate500)
- chevron: LucideIcons.chevronRight, 16px, slate400
- 高さ: 56px
- タップで詳細シート（または編集画面）
```

**空状態**:
```
         🌙
   記録がありません
  ヘルスケア連携を有効にするか、
   目覚めを記録してみましょう
```

**無限スクロール**: 30件ずつページング

### D1. MorningWakeupDialog（朝ダイアログ）

**タイプ**: Material AlertDialog（角丸 8px、背景 cardBackground）

```
┌──────────────────────────────┐
│                              │
│    ☀️ おはようございます      │  ← 24px, weight 600
│                              │
│      今朝の目覚めは？          │  ← 14px, weight 400, textSecondary
│                              │
│  ┌────────────────────────┐ │
│  │   😊   すっきり          │ │  ← 高さ56px、全幅、角丸6px
│  └────────────────────────┘ │
│  ┌────────────────────────┐ │
│  │   😐   まあまあ          │ │
│  └────────────────────────┘ │
│  ┌────────────────────────┐ │
│  │   😫   だるい            │ │
│  └────────────────────────┘ │
│                              │
│  [あとで]   [今日は聞かない]   │  ← テキストボタン2つ、下部
│                              │
└──────────────────────────────┘

パディング: 24px
ボタン間隔: 8px
オプションボタン:
  - 未選択時: 背景 slate100, 前景 textPrimary
  - ホバー/プレス: 背景 primary50
  - 絵文字は 24px, テキストは 16px weight 500
```

**インタラクション**:
- いずれかの評価を選択 → 保存 → ダイアログ閉じる → 確認トースト（「記録しました」）
- 「あとで」→ ダイアログ閉じる（次回起動時に再表示可能）
- 「今日は聞かない」→ ダイアログ閉じる（当日はもう出ない）

## 5. 画面遷移フロー

```
[ホーム画面]
  └ SleepSummaryCard タップ
      → [SleepRecordScreen]
          ├ 履歴アイテム タップ → [SleepDetailSheet]（ボトムシート、将来実装）
          └ AppBar同期ボタン → HealthKit同期実行 → スナックバー通知

  └ SleepSummaryCard（未記録時）のCTA タップ
      → [WakeupRatingSelector]（ボトムシート）
          → 選択 → 保存 → スナックバー「記録しました」

[アプリ起動時・条件一致]
  → [MorningWakeupDialog]
      ├ 評価選択 → 保存 → ダイアログ閉 → スナックバー
      ├ あとで → ダイアログ閉（リトライ可）
      └ 今日は聞かない → ダイアログ閉（当日抑止）

[設定画面]
  └ ヘルスケア連携 → [HealthSettingsScreen]
      ├ 睡眠トグル → HealthKit権限リクエスト → ON/OFF永続化
      └ 朝ダイアログトグル → ON/OFF永続化
```

## 6. アクセシビリティ・国際化

- 全テキスト日本語のみ（現状アプリは日本語専用）
- 絵文字のみに意味を依存しない: 各評価オプションは「😊 すっきり」のようにテキストを必ず併記
- タップ領域: 最小44x44pt（Appleガイドライン）
- コントラスト比: WCAG AA（textPrimary vs background で検証）
- ダークモード対応: `AppColorsExtension.dark` を使用

---

## 7. Claude Design への生成指示

以下の形式で成果物を期待します:

1. **全コンポーネント/画面のワイヤーフレーム**（低忠実度、HTMLまたは画像）
   - 各状態バリエーション（記録あり/未記録/手動のみ 等）を並列に
2. **高忠実度モックアップ**（iPhone 14 Pro サイズ 390×844）
   - ホーム画面（睡眠カード配置）
   - SleepRecordScreen（全セクション）
   - MorningWakeupDialog
   - HealthSettingsScreen（拡張部分）
3. **ダークモード版**も並べて出力
4. **SleepStageChart の案A/案B 比較図**

Flutter 実装を意識して、Material 3 のコンポーネントを忠実に再現すること。
