# FIT-CONNECT Mobile プロジェクト概要

> 最終更新: 2026-02-21 | バージョン: 1.0.0（開発中）

## 1. プロジェクト概要

**FIT-CONNECT Mobile** は、パーソナルトレーナーとクライアントをつなぐフィットネス管理アプリです。

クライアントはトレーナーとのメッセージのやり取りの中で、体重・食事・運動の記録を自動的に蓄積できます。専用の入力画面を開く必要がなく、日常のコミュニケーションがそのまま記録になる点が最大の特徴です。

## 2. アプリのコンセプト

### 「メッセージで記録が完結する」

従来のフィットネスアプリでは、体重や食事を記録するために専用の画面を開き、フォームに入力する必要がありました。

FIT-CONNECT では、トレーナーへのメッセージにタグ（例: `#体重 65.5kg`）を付けるだけで、自動的に記録が作成されます。

```
従来のフロー:
  記録画面を開く → 数値を入力 → 保存 → トレーナーに報告

FIT-CONNECT のフロー:
  トレーナーに「#体重 65.5kg 今日も頑張りました！」と送信
  → 体重記録が自動作成 + トレーナーにメッセージが届く
```

この仕組みにより、報告と記録が一つのアクションで完結します。

## 3. 対象ユーザー

### クライアント（本アプリの利用者）

パーソナルトレーニングを受けている方が対象です。主な利用シーン：

- トレーナーとの日々のやり取り（食事報告、トレーニング相談）
- 体重・食事・運動の記録と振り返り
- 目標に対する進捗の確認
- トレーナーが作成したカルテの閲覧

### トレーナー（別アプリ）

トレーナーは **FIT-CONNECT Web**（別リポジトリ）を使用します。Web側では以下を行います：

- クライアントの招待（QRコード / 招待コード発行）
- メッセージの送受信
- クライアントの記録閲覧・分析
- 目標の設定・更新
- カルテの作成・共有
- 稼働スケジュールの管理

## 5. 実装済み機能一覧

### 5.1 認証・登録フロー

トレーナーからの招待を起点とした登録フローです。

**できること：**
- QRコードスキャンまたは招待コード手動入力でトレーナーと紐付け
- トレーナー情報の確認画面で間違いがないか確認
- メールアドレスによるマジックリンク認証（パスワード不要）
- 初回登録時に名前・年齢・性別・プロフィール画像を一括設定
- 登録完了時のお祝いアニメーション表示

**登録の流れ：**
```
ようこそ画面 → QRスキャン or 招待コード入力
  → トレーナー確認 → メールアドレス入力
  → マジックリンク認証 → プロフィール設定
  → 登録完了（紙吹雪アニメーション）
```

### 5.2 ホーム画面

アプリ起動後に最初に表示される画面です。今日の状況を一目で把握できます。

**できること：**
- 日付と挨拶メッセージの表示（「こんにちは、○○さん」）
- 目標体重への進捗カード（達成率バー、残り体重、期限表示）
  - 目標達成済みの場合はゴールド背景にトロフィーアイコンを表示
- トレーナーの稼働状態カード（スケジュールに基づくオンライン/オフライン表示）
- 今日のサマリーカード：
  - 今日の食事記録数（例: 2/3食）
  - 今週の運動日数（例: 3/7日）
  - 最新体重と前日比
  - 各行タップで該当する記録タブへ遷移

### 5.3 メッセージ機能

トレーナーとの1対1チャット機能です。LINEに近い操作感を実現しています。

**できること：**
- テキストメッセージの送受信
- 画像の添付送信（最大3枚、カメラ撮影またはギャラリーから選択）
- リアルタイムでのメッセージ受信（画面を開いていれば即座に反映）
- メッセージへの返信（引用付き）
- メッセージの編集（送信後5分以内）
- 既読表示（「既読 ・ 14:30」のLINEスタイル）
- 編集済みバッジの表示
- 画像タップでフルスクリーン表示
- タグ入力時の候補表示（`#` を入力すると候補が出現）
- 上スクロールによる過去メッセージの自動読み込み（30件ずつ）

### 5.4 体重記録

メッセージのタグから自動作成された体重記録を閲覧・分析できます。

**できること：**
- 期間フィルタで表示範囲を切り替え（今週 / 今月 / 3ヶ月 / 全期間）
- 統計情報の表示：
  - 現在の体重、目標体重、残り（または超過）体重
  - 前回比、開始時からの変化、達成率
- 期間ごとの統計（平均・最高・最低・変動幅）
- 折れ線グラフによる体重推移の可視化（目標体重の点線表示付き）
- データポイントをタップすると日付と数値のツールチップを表示
- 記録一覧の時系列表示

### 5.5 食事記録

メッセージのタグから自動作成された食事記録を閲覧できます。

**できること：**
- 期間フィルタ（今日 / 今週 / 今月 / 全期間）
- 本日のサマリー（今日の食事回数とプログレスバー）
- 週カレンダー（各曜日の食事回数を表示）
- 月カレンダー（GitHub風の緑の濃淡で食事回数を可視化）
- 食事カード（食事タイプ・写真・メモの表示）
- 写真タップでフルスクリーン表示
- 食事タイプ：朝食 / 昼食 / 夕食 / 間食

### 5.6 運動記録

メッセージのタグから自動作成された運動記録を閲覧できます。

**できること：**
- 期間フィルタ（今週 / 今月 / 全期間）
- 運動タイプフィルタ（すべて / 筋トレ / 有酸素）
- 週カレンダー（曜日ごとに筋トレ💪・有酸素🏃アイコン表示）
- 月カレンダー
- 統計サマリー（総回数・タイプ別回数・合計消費カロリー）
- 記録一覧の表示

### 5.7 目標管理

トレーナーが設定した目標に対する進捗を確認できます。

**できること：**
- ホーム画面の目標カードで達成率・残り体重・期限を常時表示
- 体重記録時に自動で目標達成を判定
- 目標達成時にお祝いオーバーレイ表示（紙吹雪アニメーション付き）
- 減量・増量の両方向に対応

> 目標の設定・変更はトレーナー側Webアプリから行います。

### 5.8 カルテ（クライアントノート）閲覧

トレーナーが作成・共有したカルテを閲覧できます。

**できること：**
- 共有済みカルテの一覧表示（セッション番号・タイトル・日付・本文プレビュー）
- カルテ詳細の閲覧（全文・添付ファイル）
- 添付画像のフルスクリーン表示
- 添付PDFの外部アプリでの表示
- ファイル数バッジによる添付有無の確認

> カルテの作成・編集はトレーナー側Webアプリから行います。

### 5.9 設定

プロフィールの編集とアカウント操作ができます。

**できること：**
- プロフィール画像の変更（カメラ撮影 / ギャラリーから選択）
- クライアント名の編集
- 担当トレーナー名の確認
- ログアウト（確認ダイアログ付き）

### 5.10 プッシュ通知

新着メッセージをプッシュ通知で受け取れます。

**できること：**
- 新着メッセージのプッシュ通知受信
- フォアグラウンド時のローカル通知表示
- 通知タップでアプリを開く
- FCMトークンの自動更新

**プラットフォーム別の状況：**
| プラットフォーム | 状況 |
|----------------|------|
| Android | 実装完了・動作可能 |
| iOS | 実装完了・APNs設定待ち（Apple Developer Program登録後に完了予定） |

### 5.11 バックエンド・インフラ

#### データベース

PostgreSQL（Supabase）上に以下の主要テーブルがあります。

| テーブル | 用途 |
|---------|------|
| `trainers` | トレーナー情報（名前、メール、プロフィール画像） |
| `clients` | クライアント情報（名前、年齢、性別、目標体重、トレーナー紐付け） |
| `messages` | トレーナー・クライアント間のメッセージ |
| `weight_records` | 体重記録 |
| `meal_records` | 食事記録 |
| `exercise_records` | 運動記録 |
| `client_notes` | カルテ（トレーナーが作成、クライアントが閲覧） |
| `trainer_schedules` | トレーナーの稼働スケジュール |

#### セキュリティ

全テーブルにRow Level Security（RLS）ポリシーを設定済みです。

- クライアントは自分のデータのみアクセス可能
- メッセージは当事者（送信者・受信者）のみ閲覧可能
- メッセージ編集は送信後5分以内のみ許可
- カルテは共有フラグが有効なもののみ閲覧可能

#### Edge Functions

| 関数名 | 役割 |
|--------|------|
| `parse-message-tags` | メッセージ内のタグを解析し、体重・食事・運動の記録を自動作成。プッシュ通知の送信も担当。 |

#### ストレージ

| バケット | 用途 |
|---------|------|
| `message-photos` | メッセージに添付された画像 |
| `client-avatars` | クライアントのプロフィール画像 |

## 6. コア機能：タグによる記録自動作成

本アプリの最大の特徴である、メッセージタグによる記録自動作成の仕組みを説明します。

### タグの書式

メッセージ本文にタグを含めると、対応する記録が自動的に作成されます。

| タグ | 記録タイプ | 例 |
|------|-----------|-----|
| `#体重 ○○kg` | 体重記録 | `#体重 65.2kg` |
| `#食事:朝食` | 食事記録（朝食） | `#食事:朝食 オートミールとヨーグルト` |
| `#食事:昼食` | 食事記録（昼食） | `#食事:昼食 サラダチキンと玄米` |
| `#食事:夕食` | 食事記録（夕食） | `#食事:夕食 鶏胸肉のグリル` |
| `#食事:間食` | 食事記録（間食） | `#食事:間食 プロテインバー` |
| `#運動:筋トレ` | 運動記録（筋トレ） | `#運動:筋トレ ベンチプレス 60kg×10回` |
| `#運動:有酸素` | 運動記録（有酸素） | `#運動:有酸素 ランニング 5km 30分` |

### 入力補助

`#` を入力すると、タグの候補が自動的に表示されます。

```
「#」を入力 → [#食事] [#運動] [#体重] が候補表示
「#食」を入力 → [#食事:朝食] [#食事:昼食] [#食事:夕食] [#食事:間食]
「#運」を入力 → [#運動:筋トレ] [#運動:有酸素]
```

### 自動作成のフロー

```
① クライアントがタグ付きメッセージを送信
   例:「#体重 65.2kg 昨日より減りました！」

② メッセージがデータベースに保存される（messages テーブル）

③ データベースのWebhookトリガーが起動

④ Edge Function「parse-message-tags」が実行される
   ├── タグを解析（#体重 → 体重カテゴリ、65.2 → 数値抽出）
   ├── 対応するテーブルにレコードを作成（weight_records）
   ├── 体重記録の場合 → 目標達成判定を実行
   │   └── 達成時 → クライアントにお祝い通知を送信
   └── トレーナーに新着メッセージ通知を送信

⑤ クライアントの記録画面に自動的に反映される
```

### メッセージ編集時の動作

メッセージを編集した場合（送信後5分以内）、タグの内容が再解析されます。

- 元の記録は削除され、新しいタグ内容で記録が再作成されます
- タグを削除した場合、対応する記録も削除されます

## 8. 用語集

| 用語 | 説明 |
|------|------|
| クライアント | パーソナルトレーニングを受けている利用者のこと |
| トレーナー | パーソナルトレーナー。Web版アプリから操作する |
| タグ | メッセージに含める `#体重` `#食事:朝食` などの記号。記録の自動作成に使われる |
| マジックリンク | パスワードの代わりにメールで届くリンクをクリックして認証する方式 |
| 招待コード | トレーナーが発行する登録用のコード。QRコードとしても提供される |
| カルテ | トレーナーが作成するセッションごとの記録。共有設定されたものをクライアントが閲覧できる |
| RLS | Row Level Security。データベースの行単位のアクセス制御。自分のデータだけが見える仕組み |
| Edge Function | サーバー上で動くプログラム。タグ解析や通知送信に使われる |
| FCM | Firebase Cloud Messaging。プッシュ通知を送る仕組み |
| Realtime | Supabaseのリアルタイム通信機能。メッセージの即時反映に使われる |
| 達成率 | 開始時体重から目標体重までの進捗を0〜100%で表したもの |
| 草カレンダー | GitHubの活動グラフのように、色の濃淡で活動量を表現するカレンダー表示 |
