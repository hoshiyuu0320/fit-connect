# FIT-CONNECT Web: ワークアウトプラン管理機能 実装要件定義書

## 1. 概要
本ドキュメントは、Next.js 15 (App Router) および Supabase を用いた FIT-CONNECT Web アプリケーション における、「ワークアウトプラン管理機能」の実装要件を定義する。
現場のパーソナルトレーナーが、直感的な操作でクライアントのプラン作成・スケジュール管理・進捗把握を行えるUI/UXを実現する。

## 2. 前提技術スタック
- Framework: Next.js 15 (App Router)
- Backend/DB: Supabase (PostgreSQL)
- State Management: Zustand
- Styling: Tailwind CSS + Radix UI
- Drag & Drop Library: `@dnd-kit` (推奨)

## 3. 追加データベース設計 (Supabase)
既存のスキーマに対し、以下のテーブルを追加・連携する。

- `workout_templates`: トレーナーが作成する種目やプランのひな形。
- `workout_assignments`: クライアントに割り当てた特定の日付のプラン（宿題）。
  - カラム例: `id`, `client_id`, `trainer_id`, `assigned_date`, `status` (enum: 'pending', 'partial', 'completed', 'skipped')
- `workout_exercises`: プランに紐づく個別種目。
  - カラム例: `id`, `assignment_id`, `exercise_name`, `target_weight`, `target_reps`, `target_sets`, `order_index`

### B. 進捗ステータスバッジの可視化
- **ダッシュボード (`/dashboard`):**
  - 既存のアラートリスト に、「今週のプラン未実施アラート」を追加。
- **顧客一覧画面 (`/clients`):**
  - 各顧客カード に、今週のプラン実施状況を示すステータスバッジを追加。
  - バッジ要件: `🟢 完了`、`🟡 一部完了`、`⚪️ 未実施` (workout_assignments の status に依存)。

### C. 対面セッション用 記録UI (軸1)
- **顧客詳細画面 (`/clients/[client_id]`):**
  - 既存の6タブ（サマリー、体重、食事、運動、カルテ、チケット） に、「セッション実行」タブを追加。
  - 全体的なUI
    - ジムフロアでの操作（タブレット等）を想定し、タップ領域の広い「＋ / －」ボタンで重量・回数を入力できるUI。
  - 機能1
    - 内容
      - セット数を入力する欄のすぐ横（または薄い文字で背景）に、「前回: 60kg x 10」と表示されているUI。
    - UX
      - トレーナーはいちいち過去のページに戻る必要がありません。「前回60kgだったから、今日は62.5kgに挑戦しましょう」と即座に提案できます。
  - 機能2
    - 内容
      - セット数を入力する欄のすぐ横（または薄い文字で背景）に、「前回: 60kg x 10」と表示されているUI。
    - UX
      - トレーナーはいちいち過去のページに戻る必要がありません。「前回60kgだったから、今日は62.5kgに挑戦しましょう」と即座に提案できます。
  - 機能3
    - 内容  
      - 2つの種目をドラッグして「リンク（連結）」させる機能。
    - UX
      - インターバルなしで2種目行う場合、リスト上で2つのカードが「ガチャン」とくっつくようなUIだと、視覚的にセット構成が理解しやすくなります。
