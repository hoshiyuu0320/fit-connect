# FIT-CONNECT 機能アイデア・バックログ

> 作成日: 2026-04-29
> 出典: トレーナーアプリの雑メモを整理したもの。優先度・実装可否は未確定。

## プロダクトビジョン

- 「このアプリがないと仕事にならない、効率が落ちる」と感じてもらえるレベルの依存度
- 「トレーナー始めるなら FIT-CONNECT だよね」と言われる存在になる

---

## 1. AI機能

| 機能 | 概要 | 補足 |
| ---- | ---- | ---- |
| メッセージ返信候補 | AIが顧客への返信案を自動生成 | トレーナーの負担軽減 |
| 独自スタンプ生成 | AIによるカスタムスタンプ作成 | エンゲージメント向上 |
| ワークアウトプラン自動生成 | セッションの未定項目をAIに作らせる | 種別はセッション/配信用で分ける |
| 解説動画生成 | 宿題ワークアウト用の動画をAIが生成 | 顧客の自宅トレーニング支援 |
| AI相談機能 | 顧客の統計データを一部渡してAIに相談 | トレーナーの判断補助 |
| 食事アプリのスクショ画像分析 | あすけん等の統計画面スクショから記録を自動作成 | 他アプリユーザー流入導線にもなる ／ → `IMPLEMENTATION_TASKS.md` フェーズ2「2.5」に昇格（2026-04-29） |

## 2. スケジュール・ステータス管理（トレーナー）

- **オフの日設定** → 顧客アプリのホーム画面にトレーナーステータスを表示
  - 例: 「今日はオフなので返信が遅くなるかも」
- **週初め/月初めに休み一括入力**できるUI（入力しやすさ重視）
- **ドラッグ&ドロップでセッション設定**
  - 顧客に紐づくチケット選択ダイアログを表示
  - スケジュール画面に自動反映

## 3. 身体・健康データ記録

- **体の写真記録**
  - 将来的に3Dスキャンで統計データ化（面白い差別化要素）
- **睡眠データ**（ヘルスケア連携）
- **連続記録（ストリーク）**: 食事・運動・体重
- **期間選択での統計表示**: 平均・最大値など、全期間〜1年間
- **体重平均（1週間）**
- **カロリー収支からの体重予測**
  - ※既存企画 `memory/project_weight_prediction.md` と統合する

## 4. ワークアウト管理

- **種別分け**: セッション用 / 配信用
- **直感的UI**
  - 顧客のセッション画面はトグルで大きく分ける
  - 大きなボタンで操作しやすく
  - 縦画面・スマホ対応
- **宿題ワークアウトに解説・動画添付**
- **繰越問題の解決** ⚠️
  - 現状: 顧客が翌日にトレーニングを回すとプランが消える
  - 要件: 翌日以降に繰越できる設計が必要

## 5. コミュニケーション

- **顧客のメッセージ → トレーナーのLINEにも通知**
  - FIT-CONNECT へのリンク付きメッセージ
- **ユーザーコミュニティ機能**

## 6. SNS連携・販促

- **顧客の変化・統計データをSNSへアップロード**
  - 対応: X / Instagram / Facebook
  - 投稿内容・コメントは任意で編集可能
  - 効果: トレーナーの販促＋アプリ自体の販促
- **招待コードで割引・無料化**

## 7. 課金・マネタイズ（Stripe導入）

- **Stripe連携による課金制度の導入**
  - 想定される課金対象: トレーナーアカウント（サブスクリプション）
  - プラン設計: フリー / 有料プラン（Tier分け要検討）
  - 招待コードによる割引・無料化（セクション6と連動）
- **要検討事項**
  - 課金単位: トレーナー単位 or 顧客数ベースの従量課金
  - 決済フロー: Web/Mobile 両対応（Mobileは Apple/Google 課金規約への配慮が必要）
  - 法務要件: 特商法表記、利用規約、領収書発行
  - キャンセル・返金ポリシー

## 8. その他機能

- **トレーニング記録アプリとの連携**

## 整理が必要な論点

1. **AI機能の優先度付け**: 返信候補 / スタンプ / 動画生成 / プラン生成 / 相談 のうちどこから着手するか
2. **カルテ手書き入力**: 必要性の精査（タップ入力で代替可能か）
3. **ワークアウト繰越**: 既存設計の不具合寄り → 要件定義から詰める必要あり
4. **体重予測との統合**: 既存メモ `project_weight_prediction.md` の内容を本バックログに反映
5. **課金モデルの設計**: Stripe導入時のプラン構成・課金単位・モバイル決済規約への対応方針
