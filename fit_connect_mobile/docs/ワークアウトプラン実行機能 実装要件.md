# FIT-CONNECT Mobile: ワークアウトプラン実行機能 実装要件定義書

## 1. 概要
本ドキュメントは、Flutter および Riverpod を用いた FIT-CONNECT Mobile アプリケーション における、「ワークアウトプラン実行機能」の実装要件を定義する。
クライアント（顧客）が、トレーナーから配信された宿題プランを迷わず実行でき、完了報告が既存のメッセージベースの記録システム とシームレスに連動する仕組みを構築する。

## 2. 前提技術スタック
- Framework: Flutter (Dart)
- State Management: Riverpod
- Backend: Supabase (PostgreSQL, Realtime)
- Push Notifications: Firebase Cloud Messaging (FCM)

## 3. 実装機能要件

### A. 「プラン」タブの新設
- ボトムナビゲーションに新しいタブ「プラン（または宿題）」を追加。
- 本日割り当てられている `workout_assignments` のデータを Supabase からフェッチして表示。

### B. 実行モードとビジュアルカードUI
- **カードUI:** 種目名、目標セット数・回数に加え、視覚的にわかりやすいプレースホルダー画像（またはGIF動画サムネイル）を表示。
- **プログレス管理:** - 種目ごとにチェックボックスを配置。タップするとカードがグレーアウトし、完了状態になる。
  - 画面上部に進行度を示すプログレスバーを配置（例: 3/5種目完了）。

### C. 完了報告とタグシステムの連携（最重要）
FIT-CONNECT Mobile のコア機能である「Edge Functions（`parse-message-tags`）によるタグ解析」を再利用する。
- **機能:**
  - 全ての種目が完了し、ユーザーが「完了報告」ボタンをタップした際、バックグラウンドでトレーナー宛のチャットにメッセージを自動送信する。
  - 送信メッセージフォーマット例: `#運動:完了 本日のワークアウトプランを達成しました！`
- **効果:**
  - これにより、既存の `messages` テーブルへのインサートトリガーが発火し、Webhook経由でプッシュ通知 (FCM) がトレーナーに届く既存フロー にそのまま乗せることができる。

### D. モチベーション演出
- プラン完了時に、既存の「目標達成時のお祝いオーバーレイ表示（紙吹雪アニメーション）」 を流用し、達成感を演出する。
